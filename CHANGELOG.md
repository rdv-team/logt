### Changelog

## 2.1.0 — 30.01.2026

### Added
- Добавлена таблица calls для хранения событий CALL технологического журнала в ClickHouse
- Добавлено извлечение и сохранение метрик из события CALL (CpuTime, Memory, MemoryPeak)
- Добавлено связывание событий CALL с операциями VRSREQUEST-VRSRESPONSE и вывод события CALL в обработке "Трассировка длительных операций"
- Добавлен вывод CpuTime, Memory, MemoryPeak в обработке "Трассировка длительных операций"
- Добавлен вывод в отдельных полях итогов с общей длительностью событий анализируемой операции без учета CALL и SESN
- Добавлено расширение 1С RDV_LogT версии 2.1.0 (releases_1c)

### Changed
- Обновлено руководство пользователя: описание показателей CALL и новый скриншот интерфейса обработки
- Оптимизирован интерфейс обработки "Трассировка длительных операций"

## 2.0.0 — 12.12.2025

### Added
- Добавлено расширение трассировки операций версии 2.0.0
- Добавлена трассировка фоновых заданий (BackgroundJob): в результат обработки теперь попадут не только пользовательские операции, но и фоновые задания и события ТЖ, которые эти фоновые задания вызывали.
- Возможность выбора в обработке, собирать ли клиент-серверные (пользовательские) операции или фоновые задания
- Сохранение в событиях и операциях свойства t:connectID для упрощения поиска виновников блокировок по данным WaitConnections. Отбор по connectID доступен в обработке 1С.
- В обработке 1С на вкладе "Последняя строка контекстов" добавлена информация о полном стеке вызовов этой последней строки. Это помогает понять откуда вызывалась последняя строка контекта и какой стек занял самое длительное время.
- В интерфейсе обработки трассировки добавлены колонки, показывающие наличие событий TTIMEOUT, TDEADLOCK, EXCP и другие системные события
- В README добавлена инструкция по легкой и быстрой установке ClickHouse на ОС Windows
 
### Changed
- Обновлено руководство пользователя
- Процентное распределение событий теперь считается относительно общей длительности всей операции
- Дополнена структура таблиц ClickHouse полем connect_id (см. create_table_clickhouse.sql)
- Повышение удобства интерфейса обработки
- Исправлены ошибки отборов за период по дате операции и установка лимита
- Оптимизирована трассировка пользовательских операций
